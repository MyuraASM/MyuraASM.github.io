File System

PussyOS has its own in memory filesystem. Everything lives in RAM while the OS runs files won't survive a reboot unless you save them to disk with savefs.

# basic workflow
> mkdir projects
> cd projects
> mkfile notes.txt
> write notes.txt hello world
> cat notes.txt
hello world

# save everything to disk
> savefs
Saving filesystem to LBA 100... saved successfully!
Note: Both savefs and loadfs default to LBA 100. Pass a number to use a different sector e.g. savefs 200.

File Transfer

Send and receive files between PussyOS and the host machine over the network. Requires a working E1000 network connection.

Receiving

Run recvfile to start listening, then have the host send the file. Run stoprecvfile when done.

> recvfile
Listening for incoming file...
# host sends the file over the network
> stoprecvfile

Sending

Use sendfile with the name of a file that already exists in the filesystem.

> sendfile notes.txt

Text Editor

The edit command opens a fullscreen text editor. If the file doesn't exist it gets created automatically.

> edit notes.txt

Type freely, navigate with arrow keys, delete with Backspace. Save and exit with the key combo shown at the bottom of the editor.

edit vs write: Use edit for interactive writing, use write for quick one liners or inside scripts.

Networking

PussyOS has real networking via an Intel E1000 card. The stack handles ARP, TCP, and UDP from scratch. The card initializes automatically at boot. If it's not detected, all network commands fail run pciscan to check. Configure your IP and MAC with netsettings before connecting to anything.

netsettings

Fullscreen panel to view and edit your IP, MAC, gateway IP, and gateway MAC. Arrow keys to select a field, Enter to edit, ESC to exit.

> netsettings
Field What it is
My IP The IP address PussyOS uses on the network.
Gateway IP Your router. Required for reaching outside the subnet.
My MAC Hardware address of the E1000 card.
Gateway MAC Router MAC. Auto resolved via ARP when you connect, or set manually.

NetPussy np

Raw TCP client. Connects to any IP and port, then shows a menu of premade payloads you can fire at the target. Press R to switch to raw input mode and type your own data.

> np 192.168.0.1 80
# arrow keys to pick payload, Enter to send, R for raw mode, ESC to disconnect
Category Payloads
SMB SMB negotiate (v1 detect), SMB2 negotiate
HTTP GET /, GET example.com, OPTIONS /
FTP Anonymous login
Mail SMTP EHLO, POP3 CAPA, IMAP CAPABILITY
Redis PING, INFO dump, CONFIG GET *
Other MongoDB isMaster, Elasticsearch info, WinRM check, null probe, raw input

Port Scanner npps

Scans a TCP port range on a target and lists open ports with service names where known. Progress bar updates live. ESC aborts mid scan, arrow keys scroll results after.

> npps 192.168.0.1 1 1024

Chat

UDP chat on port 9999 between PussyOS and the host. The host sends first PussyOS picks up that address and uses it for replies from then on.

> chat
# waiting for host to send first, then type and Enter to reply, ESC to exit

Web Server webserver

TCP server on port 80 with a fullscreen HTML editor. Write your page, Ctrl+D to deploy it live, Ctrl+S to save to the filesystem, Ctrl+X to exit. A request counter in the title bar shows hits.

> webserver
> webserver mypage.html # load existing file
Note: The page is not live until you hit Ctrl+D. The status bar at the bottom tells you if you're deployed or not.

Network Monitor netmon

Live packet monitor. Prints all incoming traffic to the screen in real time useful for debugging ARP, UDP, and TCP activity. Beta, no filtering yet. ESC to stop.

> netmon

Brainfuck

PussyOS runs a numeric variant of Brainfuck. Instead of the classic symbols, you use digits 0 7. Write your script to a file and run it with bf.

> bf hello.bf

Instruction Set

Digit Classic BF What it does
0 > Move pointer right
1 < Move pointer left
2 + Increment current cell
3 Decrement current cell
4 . Output current cell as ASCII
5 , Read one keypress into current cell
6 [ Jump past matching 7 if cell is 0
7 ] Jump back to matching 6 if cell is not 0

Limits

Thing Limit
Tape size 30,000 cells wraps at both ends
Max code length 8,192 characters
Max loop depth 256
Max iterations 10,000,000 then stops with a warning

Anything that isn't 0 7 is ignored, so you can write comments freely in the same file.

# set cell to 65 (ASCII 'A') then output it
222222222222222222222222222222222222222222222222222222222222222224
A
Note: Brackets are validated before execution starts. A mismatched 6 or 7 errors out immediately and tells you the position.

Pussy Lang

Pussy Lang is PussyOS's own scripting language. Files use the .p or .pussy extension and run with pussy. Scripts open in an interactive fullscreen viewer output scrolls live, ESC exits. When the script finishes, output is saved to output.txt automatically (or wherever you point outfile).

> pussy myscript.p
Keys while running: Up/Down to scroll output. ESC to exit.

Variables

Variables are referenced with $name. No declaration needed just set them.

Command Syntax Description
set set $x 42 Set a variable. Works with numbers and strings.
add add $x 10 Add to a numeric variable in place.
sub sub $x 5 Subtract from a numeric variable in place.

Output

Command Description
say <text> Print a line. Expands $vars inside strings.
sayinline <text> Same but no newline.
outfile <name> Set the output save file. Default: output.txt.
set $name "world"
say "hello $name"
hello world

Input

Command Description

getkey $var Read the last pressed key's scancode (non blocking).
waitkey $var Block until a key is pressed, store its scancode.

Control Flow

Command Syntax Description
if if $x == 5 { ... } Run block if condition is true. Operators: == != < > <= >=
loop loop 10 { ... } Repeat a block N times.
while while $x < 10 { ... } Loop until condition is false. Capped at 10,000 iterations.
set $i 0
while $i < 3 {
  say "tick $i"
  add $i 1
}
tick 0
tick 1
tick 2

Functions

Define with func, call with call. Return values land in $ret. Variables are scoped per call saved and restored automatically. Max call depth: 50.

func greet $who {
  say "hello $who"
  return "done"
}

call greet "pussy"
hello pussy
